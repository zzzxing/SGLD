# AGENTS.md

## 项目身份
本仓库项目是“追问智学”：一个面向初中信息技术课堂的苏格拉底教学法智能辅学 Web 系统。

## 总体目标
构建一个可运行、可迭代、可在 Windows 局域网环境部署的正式工程。
学生端为主，教师端为辅，管理员端需要具备实际管理能力。
不做模型微调，只做现有模型 API 接入 + 提示词工程 + 轻量 RAG + 教学流程控制。

## 技术栈约束
- Python 3.11
- FastAPI
- Jinja2
- HTMX
- 少量原生 JavaScript
- WebSocket
- SQLAlchemy
- SQLite 优先，未来兼容 MySQL
- 本地文件存储
- Windows 局域网部署优先
- 必须支持 bat 一键启动

## 架构原则
- 优先单体 Web + 模块化
- 优先可运行，再补强，再美化
- 不要无意义前后端分离
- 不要引入维护成本过高的重型框架
- 不要把业务逻辑堆到单个文件
- 目录结构要清晰，便于小白维护

## 开发优先级
优先先做“可跑通的纵向切片”：
1. 项目能启动
2. 基础配置与目录结构清晰
3. 登录与角色区分（管理员/教师/学生）
4. 学生端课程/章节列表与学习入口
5. 教师端课程/章节管理基础页
6. 管理员端模型配置基础页
7. 文件上传入口
8. 模型适配层骨架
9. 提示词工程配置骨架
10. bat 一键启动脚本
11. README/运行说明

随后再做：
12. 学生端 AI 对话页
13. 问题链/学习导航
14. 教材解析与轻量 RAG
15. Python 代码输入与运行
16. 教师端课堂控制
17. 课堂统计分析
18. 评分、评语、日志、安全约束

## 教学与产品要求
- 系统面向初中信息技术课堂
- AI 以苏格拉底式追问为核心，不应默认直接给答案
- 支持分层提示：先追问，再提示，最后必要时给答案与总结
- 默认围绕当前章节、当前知识点、当前任务展开
- 学生端要有轻量学习导航：当前知识点、当前步骤、为何继续追问
- 不要把提示词链路完整暴露给学生

## 多模型与 RAG 要求
- 必须做统一模型适配层，不写死单一提供方
- 支持配置 provider、model、base_url、api_key、timeout、retry
- 采用“轻量 RAG + 提示词工程”
- 资料检索顺序：当前章节优先、当前任务优先、当前知识点优先

## Python 代码练习要求
- 支持代码输入或粘贴
- 支持服务器端安全运行 Python 代码
- 返回运行结果与报错
- AI 基于代码和报错进行苏格拉底式分析引导
- 不做过重在线 IDE

## 安全要求
- 对话默认限定在课堂学习范围
- 对明显不适合课堂的内容做基础拦截
- 上传文件类型和大小要限制
- 代码运行环境要做隔离和限制
- 关键操作要校验权限
- 日志要保留必要信息

## 代码风格要求
- 命名统一、模块清晰、注释适度
- 配置集中管理
- 错误处理明确
- 路由、服务、数据层尽量分离
- 每个新增模块必须说明用途
- 优先写可读、可维护代码，不炫技

## 每轮工作要求
每轮结束时必须输出：
1. 本轮修改文件列表
2. 完成功能清单
3. 运行或验证结果
4. 发现的问题
5. 已修复的问题
6. 下一轮计划

## 自审要求
提交前必须自查：
- 是否能运行
- 是否符合需求文档
- 是否符合课堂场景
- 是否对小白维护友好
- 是否有明显安全隐患
- 是否需要补测试或文档

## 禁止事项
- 不要只生成空壳页面
- 不要只做静态原型
- 不要只做数据库不接页面
- 不要只做页面不接后端
- 不要只做代码不更新 README
- 不要做完一轮就停在“建议下一步”

## 遇到不确定时
- 先选更稳妥、更易维护的方案
- 优先保证 Windows 和局域网可用
- 优先保证项目能运行
