## 4. 学生端核心页面设计

## 4.1 学生首页
### 模块
- 顶部导航
- 今日课堂卡片
- 当前班级信息
- 最近学习记录
- 快速进入当前任务按钮

### 目标
让学生一眼知道自己今天该做什么。

## 4.2 当前课堂页（关键页）
### 布局建议
- 左侧：章节目录 / 当前知识点导航
- 中间：知识讲解区 + 问题链对话区
- 右侧：当前进度、提示层级、任务要求、提交入口

### 功能模块
1. 当前章节标题
2. 当前知识点卡片
3. 讲解内容区
4. “开始思考”按钮
5. 问题链对话区
6. 文字作答输入框
7. 代码输入框
8. 运行代码按钮
9. 运行结果面板
10. 提交答案按钮
11. 当前步骤提示
12. 小结与完成状态

### 页面状态
- 未开始
- 进行中
- 等待输入
- 代码运行中
- AI 生成中
- 已完成

## 7. 关键流程图（文字版）

## 7.1 学生上课流程
登录 → 等待教师推送  
→ 进入课堂页  
→ 阅读讲解  
→ 接收第一问  
→ 回答  
→ 系统判断  
→ 继续追问/给提示  
→ 提交任务  
→ 完成本节小结

## 7.2 教师备课流程
上传教材  
→ 系统解析  
→ 生成章节/知识点/讲解/问题链初稿  
→ 教师修订  
→ 发布到班级

## 7.3 管理员配置流程
创建教师  
→ 创建班级  
→ 批量导入学生  
→ 配置模型  
→ 配置资源库  
→ 配置系统参数

## 8. UI 风格建议（锁定版）

你已经明确希望偏**现代 AI 产品风、卡片化、轻科技感**。  
因此页面风格建议如下：

### 视觉关键词
- 深浅对比明确
- 大面积留白
- 圆角卡片
- 弱玻璃感或半透明浮层
- 清晰的一级、二级信息层级
- 蓝紫 / 青蓝系科技感点缀
- 但不做过度动画

### 学生端
- 重点突出“当前任务”和“当前步骤”
- 卡片化讲解
- 对话区域要像 AI 产品，但比聊天软件更克制

### 教师端
- 像现代控制台
- 重状态概览和效率

### 管理员端
- 简洁后台风
- 少装饰，多可控项
