# Codex 开发任务书（可直接投喂）

## 1. 任务目标

请开发一个名为“追问智学”的局域网 Web 教学系统。  
该系统面向**初中信息技术课堂**，运行在**Windows 电脑局域网服务器**上，支持**管理员端、教师端、学生端**三种角色，核心交互是**苏格拉底式追问辅学**。

不要做模型微调。  
请使用**现成国内大模型 API**，采用**多模型可切换**架构。  
请以**Python 3.11 + FastAPI + Jinja2/HTMX + SQLAlchemy + WebSocket**为第一版技术路线。

## 2. 项目必须满足的核心要求

### 2.1 角色
- 管理员端：管理教师、班级、学生、模型 API、资源库、系统设置
- 教师端：上传教材、自动生成章节/知识点/讲解/问题链、布置课堂、看实时进度、点评学生
- 学生端：查看讲解、按问题链学习、提交文字/代码/文件、接收追问和提示

### 2.2 课堂场景
- 一个班在机房同时使用
- 教师可以一键推送全班进入某章节/任务
- 学生端页面实时切换或收到提示
- 教师端可以实时看到全班进度、在线状态、卡点分布

### 2.3 学习流程
学生进入课堂页后：
1. 先看知识讲解
2. 再进入问题链式对话
3. 提交文字思路 / Python 代码 / 任务结果
4. 系统按苏格拉底方式追问、给提示、必要时给答案与总结

### 2.4 内容生成与管理
- 教师上传教材/讲义
- 系统自动切分章节与小节
- 自动提取知识点
- 自动生成知识讲解初稿
- 自动生成问题链初稿
- 自动生成任务/练习初稿
- 教师可以修改并发布

### 2.5 技术限制
- 以 Windows 本机/局域网部署为主
- 要能一键启动
- 不要要求用户必须懂 Docker
- 不要做复杂前后端分离

## 3. 推荐技术栈（请遵守）

## 8. LLM 设计要求

### 统一适配接口
请封装统一 LLM Provider 接口，能够兼容多个国内模型供应商。

必须支持：
- 按 provider + model 配置调用
- 支持 chat
- 支持 embeddings（若供应商可用）
- 支持超时、重试、失败降级

### 必做
- 模型注册表
- 模型路由器
- 默认模型策略
- 备用模型策略

## 9. 提示词工程要求
