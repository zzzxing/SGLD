# 部署与一键启动方案（Windows 局域网版）

## 1. 目标

你的第一版运行目标是：

- 在 Windows 电脑上运行
- 这台电脑作为局域网服务器
- 机房其他电脑通过浏览器访问
- 尽量做到一键启动
- 不要求云服务器和 Docker 才能跑

## 2. 推荐部署路线

### 第一阶段（你当前最适合）
- Windows
- Python 3.11
- 虚拟环境 venv
- SQLite 或 MySQL
- Uvicorn 启动
- `start_server.bat` 一键启动

### 第二阶段（后续可选）
- Windows + MySQL 正式化
- Nginx 反向代理（可选）
- Docker Compose（可选）
- Linux 服务器迁移（可选）

## 3. 安装前提

你本机需要：
1. Python 3.11
2. PyCharm（已有）
3. MySQL（可选，若不用则先 SQLite）
4. 一台局域网稳定机器作为教师机/服务机

## 6. 一键启动脚本建议

### 6.1 `start_server.bat`
```bat
@echo off
cd /d %~dp0

if not exist .venv (
    python -m venv .venv
)

call .venv\Scripts\activate

python -m pip install --upgrade pip
pip install -r requirements.txt

python scripts\init_db.py
python scripts\seed_demo_data.py

uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
pause
```

### 说明
这适合开发演示环境。  
正式演示时可以拆成：
- `install_once.bat`
- `run_server.bat`

这样避免每次都重复装依赖。

## 7. 局域网访问说明

### 7.1 服务端启动方式
```bash
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

### 7.2 学生端访问方式
在学生电脑浏览器打开：
```text
http://教师机IP:8000
```

例如：
```text
http://192.168.1.100:8000
```

## 8. MySQL 与 SQLite 选择建议

### SQLite
适合：
- 开发期
- 单机调试
- 快速演示

### MySQL
适合：
- 一个班同时访问
- 更稳定的局域网部署
- 正式课堂使用

### 推荐策略
- 先用 SQLite 把功能跑通
- 第二版切 MySQL

## 9. 代码运行模块部署建议

由于学生会在线运行 Python 代码，必须注意安全。

### 第一版最低要求
- 只允许运行 Python
- 每次执行创建独立临时目录
- 设置执行超时
- 限制输出长度
- 过滤危险导入
- 不允许访问上层目录
- 不允许长时间运行

### 伪代码思路
```python
subprocess.run(
    ["python", temp_file],
    capture_output=True,
    text=True,
    timeout=3,
    cwd=temp_run_dir
)
```

## 14. 部署风险提醒
1. Windows 防火墙可能拦截端口
2. 局域网 IP 会变化，建议固定教师机 IP
3. OCR 依赖安装容易出问题，第一版可不把图片解析作为主流程
4. 向量库安装要先验证
