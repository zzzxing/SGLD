# 技术架构设计

## 1. 设计目标

考虑到你目前只有：
- Windows 电脑
- Python 3.11
- PyCharm
- 计划交给 Codex 辅助开发
- 需要局域网部署
- 需要一键启动
- 不希望前后端拆得太复杂

因此第一版不建议做传统重型前后端分离，而建议采用：

## 2. 推荐技术栈（第一版）

### 2.1 总体架构
**单体 Web 架构（Monolith）**

- 后端：FastAPI
- 页面渲染：Jinja2 + HTMX + 少量 Alpine.js
- 实时通信：WebSocket
- ORM：SQLAlchemy
- 数据库：SQLite（开发）/ MySQL（正式局域网）
- 任务队列：APScheduler（轻量定时任务）+ 后续可换 Celery
- 文件存储：本地文件系统
- 向量检索：FAISS 或 Chroma（二选一，建议 FAISS）
- 代码运行：受限子进程 + 沙箱目录
- 样式：Tailwind CSS（编译版或 CDN 版）
- 图表：ECharts
- 日志：Loguru / 标准 logging
- 配置：`.env`
- Windows 启动：`.bat` 脚本

### 2.2 为什么不用“重前后端分离”
因为你的阶段目标不是做复杂工程协作，而是：
- 先把功能跑通
- 降低部署门槛
- 让 Codex 更容易一次写成
- 减少接口联调成本
- 方便 Windows 局域网启动

### 2.3 为什么用 FastAPI
- Python 生态友好
- 写 API 和 WebSocket 都顺手
- 文档与类型标注清晰
- 适合做模型编排和文件处理
- 后续若想拆前后端也方便

## 5. 关键模块说明

## 5.1 教材解析模块
职责：
- 处理 Word/PDF/PPT/TXT/MD/网页文本/图片
- 提取原始文本
- 做章节切分
- 生成知识点候选
- 存储解析结果

输出：
- 原始文本片段
- 章节树 JSON
- 知识点列表
- 可选的图片/OCR 文本

## 5.2 RAG 模块
职责：
- 将教材片段向量化
- 存储索引
- 在对话或生成讲解/问题链时做相关内容检索
- 为模型提示词拼接“课程上下文”

原则：
- 轻量、可控
- 仅在当前课程 / 当前章节范围优先检索
- 不做全库随意检索

## 5.3 模型路由模块
职责：
- 管理多个模型供应商配置
- 统一请求入口
- 根据任务类型选择模型
- 处理重试、超时、降级

任务类型示例：
- 讲解生成
- 问题链生成
- 学生对话
- 作业点评
- 教材解析辅助
- 课堂总结

## 5.4 对话编排模块
职责：
- 维护问题链状态机
- 记录学生当前步骤
- 决定下一步追问
- 调用 RAG 与模型
- 形成结构化输出

## 5.5 代码运行模块
职责：
- 接收学生 Python 代码
- 在安全目录执行
- 返回 stdout/stderr
- 控制超时与资源使用
- 将运行结果作为模型分析输入的一部分

## 5.6 实时课堂控制模块
职责：
- 教师一键开始课堂
- 推送当前章节/任务到学生端
- 更新全班进度
- 同步在线状态

## 6. 页面技术策略

### 6.1 学生端
- 以服务端渲染为主
- 局部刷新使用 HTMX
- 聊天区和进度区可用 WebSocket 增量更新

### 6.2 教师端
- 列表、表格、卡片混合布局
- 实时统计用 WebSocket 或轮询兜底

### 6.3 管理员端
- 偏后台管理页面风格
- 重功能、轻动画

## 7. 实时方案

第一版建议：
- WebSocket 负责课堂推送、在线状态、学生进度、教师看板
- 关键失败时允许页面轮询兜底
- 不依赖复杂消息队列

## 8. 模型调用架构建议

### 统一适配层
定义统一接口：

```python
class LLMProvider:
    def chat(self, messages, model, temperature, extra=None): ...
    def embeddings(self, texts, model): ...
```

### 适配器实现
- DeepSeekAdapter
- DashScopeAdapter
- QianfanAdapter
- ZhipuAdapter

### 路由器
根据“任务类型 + 优先模型 + 可用性”选择最终模型。

## 9. 代码运行安全建议

第一版必须实现：
- 仅允许运行 Python
- 仅允许标准输入输出型短程序
- 禁止危险模块
- 禁止访问系统敏感目录
- 独立运行目录
- 执行超时（如 3 秒）
- 输出长度限制
- 错误信息脱敏

## 10. 为什么不建议第一版上 Docker 作为硬要求

你当前是 Windows 小白环境，且目标是先在机房局域网跑通。  
因此第一版文档里可写“可选 Docker”，但主路径应是：

**Python 虚拟环境 + MySQL/SQLite + bat 启动脚本**

这样最容易先落地。
