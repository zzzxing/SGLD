# 数据库设计（第一版）

## 1. 设计说明

第一版建议：
- 开发环境可先用 SQLite
- 局域网正式演示建议换 MySQL
- ORM 统一使用 SQLAlchemy
- 所有表设计尽量为后续扩展预留字段

## 2. 核心实体关系

- 用户 User
- 角色 Role
- 班级 Class
- 教师班级关联 TeacherClass
- 学生班级关联 StudentClass
- 课程 Course
- 章节 Chapter
- 知识点 KnowledgePoint
- 资源 Resource
- 问题链 QuestionChain
- 问题链步骤 QuestionStep
- 任务 Task
- 课堂 Session
- 学习记录 StudyRecord
- 对话消息 DialogueMessage
- 代码提交 CodeSubmission
- 任务提交 TaskSubmission
- 教师评价 TeacherFeedback
- 模型配置 ModelProvider
- 系统配置 SystemConfig

## 4. 索引建议

### 高频索引
- users.username
- classes.name
- chapters.course_id + order_no
- knowledge_points.chapter_id + order_no
- question_steps.chain_id + step_no
- study_records.session_id + student_id
- dialogue_messages.study_record_id + created_at
- code_submissions.study_record_id + created_at
- classroom_sessions.class_id + session_status

## 5. 数据状态设计建议

### 统一状态思想
很多表建议保留：
- draft
- published
- archived
- disabled

这样方便教师编辑与回滚。

## 6. 版本化建议

需要版本化的对象：
- 章节树
- 知识点讲解
- 问题链
- 任务模板

第一版可用简单 `version_no` 实现，后续再做独立历史表。
